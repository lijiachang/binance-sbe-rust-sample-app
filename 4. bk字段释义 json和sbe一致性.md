## SBE BestBidAskStreamEvent（bookticker/BBO）字段释义

字段定义见 `stream_1_0.xml` 中 `BestBidAskStreamEvent(id=10001)`：

- **eventTime（`event_time_us`）**：事件时间，`utcTimestampUs`，单位 **微秒**（us）。表示撮合/行情系统生成该条最优买卖盘更新的时间。
- **bookUpdateId（`book_update_id`）**：订单簿更新序号，`updateId`。用于判断更新先后顺序、对齐/去重等。
- **priceExponent（`price_exponent`）**：价格指数（缩放因子），`exponent8`（i8）。`bidPrice/askPrice` 的真实价格需要用它做缩放换算。
- **qtyExponent（`qty_exponent`）**：数量指数（缩放因子），`exponent8`（i8）。`bidQty/askQty` 的真实数量需要用它做缩放换算。
- **bidPrice（`bid_price_mantissa`）**：最优买价的 **mantissa**（i64）。真实价格 = `bid_price_mantissa * 10^(price_exponent)`。
- **bidQty（`bid_qty_mantissa`）**：最优买量的 **mantissa**（i64）。真实数量 = `bid_qty_mantissa * 10^(qty_exponent)`。
- **askPrice（`ask_price_mantissa`）**：最优卖价的 **mantissa**（i64）。真实价格 = `ask_price_mantissa * 10^(price_exponent)`。
- **askQty（`ask_qty_mantissa`）**：最优卖量的 **mantissa**（i64）。真实数量 = `ask_qty_mantissa * 10^(qty_exponent)`。
- **symbol（`symbol`）**：交易对字符串，`varString8`（1 字节长度 + UTF-8 数据）。

### mantissa / exponent 换算公式

- **真实值** = \( \text{mantissa} \times 10^{\text{exponent}} \)
- 当 exponent 为负数时，等价于把小数点向左移动 \(|exponent|\) 位。

用本文件示例数据验证（`price_exponent = -8`，`qty_exponent = -8`）：

- `bid_price_mantissa = 6985702000000`  
  \(6985702000000 \times 10^{-8} = 69857.02\) → `bid_price = "69857.02"`
- `bid_qty_mantissa = 204000`  
  \(204000 \times 10^{-8} = 0.00204\) → `bid_qty = "0.00204"`
- `ask_price_mantissa = 6985703000000`  
  \(6985703000000 \times 10^{-8} = 69857.03\) → `ask_price = "69857.03"`
- `ask_qty_mantissa = 6796000`  
  \(6796000 \times 10^{-8} = 0.06796\) → `ask_qty = "0.06796"`

### 与 JSON bookTicker 字段对照

你上面那条 JSON 行情（非 SBE 的 bookTicker 风格字段）：

- `u` ↔ `book_update_id`
- `s` ↔ `symbol`
- `b` ↔ `bid_price`（SBE 中由 `bid_price_mantissa + price_exponent` 换算得到）
- `B` ↔ `bid_qty`（SBE 中由 `bid_qty_mantissa + qty_exponent` 换算得到）
- `a` ↔ `ask_price`（SBE 中由 `ask_price_mantissa + price_exponent` 换算得到）
- `A` ↔ `ask_qty`（SBE 中由 `ask_qty_mantissa + qty_exponent` 换算得到）

另外：SBE 事件里多了 `event_time_us`（微秒）这类“事件时间”字段；而你这条 JSON bookTicker 示例里没有事件时间字段（只有更新序号 `u`）。
